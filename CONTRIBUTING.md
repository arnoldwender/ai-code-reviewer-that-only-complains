# Contributing to AI Code Reviewer That Only Complains

Oh, you want to contribute? The current code reviewer persona would have a LOT to say about that decision. But sure, come on in.

## Getting Started

```bash
git clone https://github.com/arnoldwender/ai-code-reviewer-that-only-complains.git
cd ai-code-reviewer-that-only-complains
npm install
npm run dev
```

The app should be running at `http://localhost:5173`. Prepare for emotional damage.

## How to Contribute

1. **Fork the repo** and create a feature branch (`feat/new-insult-category`)
2. **Make your changes** — new personas, sharper roasts, more achievements
3. **Test locally** — make sure the complaints are actually funny
4. **Submit a PR** with a clear description (the reviewer will complain about it anyway)

## Code Style

- TypeScript, strictly typed — the code reviewer would roast us for using `any`
- React functional components with hooks
- Tailwind CSS for styling
- Roasts should be clever, not cruel — think "disappointed senior dev," not "internet bully"
- Every complaint should have a grain of truth that makes developers nod and laugh

## Ideas We Love

- New reviewer personas (the intern who just learned about clean code, the architect who hates everything)
- Language-specific complaints (Python indentation drama, JavaScript callback hell jokes)
- More achievements for surviving brutal reviews
- Complaint combos — when multiple roasts chain together

## Bug Reports

Open an issue with steps to reproduce. If the reviewer accidentally says something nice, that's a P0 bug.

## Feature Requests

Have an idea for a new way to crush a developer's spirit (lovingly)? Open an issue with the `enhancement` label.

## The Golden Rule

Punch up, not down. We're laughing WITH developers about the universal experience of code review, not AT anyone. If a roast would actually hurt someone's feelings, it doesn't belong here.
